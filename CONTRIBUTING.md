# Contributing

Thank you for helping improve `tailor-resume`.

Useful contributions include:

- Better evidence-based resume writing guidance
- Additional DOCX structural checks
- Cross-platform document conversion notes
- Examples for different role families and languages
- Bug reports with a minimal, privacy-safe reproduction

## Pull Requests

1. Keep resume claims evidence-based and avoid instructions that encourage
   fabricated experience, metrics, or proficiency.
2. Keep the main `SKILL.md` concise. Put detailed guidance in `references/`.
3. Do not commit real resumes, portraits, phone numbers, email addresses, or
   other candidate data.
4. Run the skill validator and relevant script checks before opening a PR.
5. Explain the user-facing behavior changed by the contribution.

## Privacy

Never include personal resume files or identifying candidate information in
issues, fixtures, screenshots, or pull requests. Use synthetic examples.
