# Resume Content Strategy

## Evidence Language

- **Professional use:** “负责、实施、交付、推动” only when supported by work evidence.
- **Project use:** “设计、开发、实现、测试、对比” when supported by artifacts.
- **Practice:** “完成练习、在靶场实践、使用基础功能”.
- **Knowledge:** “了解、理解、熟悉基础”.
- **Interest:** “关注、持续学习”.

Never upgrade a lower evidence level into a higher one.

## Bullet Formula

`Capability label: action + target + method/tool + verified output`

Examples:

- `漏洞闭环：整理复现步骤、测试证据与影响范围，跟进修复并参与复测。`
- `通信安全：实现会话密钥协商、流加密与完整性校验，保护协议数据传输。`
- `数据处理：清洗传感器数据，对比三类模型并记录准确率与错误分布。`

## Selection Rules

- Prefer direct role relevance over technical complexity.
- Prefer artifacts that can be inspected: code, report, GitHub, deployed product, certificate.
- Keep strong internship bullets specific but confidential.
- Include metrics only when traceable.
- A portfolio URL should be visible as text for PDF readability.
- Do not list every project. Three strong projects usually beat six shallow projects.

## Student Resume Order

Common default:

1. Header/contact/availability
2. Education
3. Relevant internship
4. Relevant projects/practice
5. Skills/certificates

Move a highly relevant project above a weak internship only when justified.

## Quality Warning Signs

- Every bullet starts with “负责”.
- Claims have no object, method, or evidence.
- Skills list is broader than the projects demonstrate.
- Self-study is presented as production experience.
- Job keywords appear in a disconnected “understanding” sentence.
- The resume is dense because weak content was not removed.
