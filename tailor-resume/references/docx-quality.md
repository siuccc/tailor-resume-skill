# DOCX Resume Quality

## Formatting System

Define and apply these roles consistently:

- Section heading
- Entry title
- Date/location
- Body text
- Bold body label
- URL
- Header name/contact

Set font family for `eastAsia`, `ascii`, and `hAnsi`. Do not rely on theme fonts.

## Entry Headings

Put title and date in one paragraph:

`Project title<TAB>YYYY.MM-YYYY.MM`

Use the same right-aligned tab stop for every school, internship, and project entry. Never align dates with repeated spaces. Shorten an overly long title before allowing the date to wrap.

## Bold Rules

- Bold: entry title and date.
- Bold: the label before a Chinese colon in body bullets.
- Regular: explanatory body text, GPA details, course list, and visible URLs.
- Section styling follows the approved template.

## One-Page Check

- Use readable body type, normally 10–11 pt.
- Remove low-value material before reducing font size.
- Keep each project to 1–3 bullets.
- Avoid orphaned headings and excessive blank space.
- Confirm the photo is proportionate and not distorted.

## Structural Audit

Check:

- no comments or tracked-change residue;
- no template placeholders;
- one intended photo;
- consistent font channels;
- consistent entry tab positions;
- no accidental bullets/numbering;
- no unintended blue/underlined body text;
- visible URLs;
- target-specific filename;
- source resume remains unchanged.

## Visual Audit

Render and inspect:

- page count and clipping;
- text overlap;
- title/date alignment;
- bold scope;
- line wrapping around long English strings;
- section hierarchy;
- density and whitespace;
- photo crop and alignment.
